This guide shows you how to create API keys in Kinde for your registered APIs.

API keys can be created at the organization level (managed by admins) or user level (managed by individual users), depending on your business model.

## Prerequisites

Before creating API keys, ensure you have:

1. **Registered your APIs** in Kinde (see [API registration guide](/developer-tools/your-apis/))
2. **Defined scopes** for your APIs (optional)
3. An **M2M application** set up for [Kinde management API](/developer-tools/kinde-api/connect-to-kinde-api/) (if you intend to use our API)

## Creating API keys

You can enable your customers to [create API keys via the self-service portal](/api-keys/api-key-setup/self-serve-api-keys/).

You can also create API keys on their behalf in the Kinde admin area or via API which is what this guide will cover.

### Via the Kinde admin area

#### Organization-level keys

1. Go to **Organizations**
2. Select an organization.
3. Select **API keys** from the left sidebar.
4. Click **Add API key**.
5. Configure the key:
   - **Name**: Give it a descriptive name (e.g., "Production Integration")
   - **API**: Select the API this key can access
   - **Scopes**: Choose specific scopes for the API
6. Select **Save**

#### User-level keys

1. Go to **Users**
2. Select a user.
3. Select **API keys** from the left sidebar.
4. Click **Add API key**.
5. Configure the key:
   - **Name**: Give it a descriptive name (e.g., "Production Integration")
   - **API**: Select the API this key can access
   - **Scopes**: Choose specific scopes for the API
6. Select **Save**

### Via API

Use your [M2M application](/developer-tools/kinde-api/connect-to-kinde-api/) to create API keys programmatically:

```bash
# Create an organization-level key
curl -X POST https://your-domain.kinde.com/api/v1/api_keys \
  -H "Authorization: Bearer YOUR_M2M_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Integration",
    "api_id": "<your-api-id>",
    "scope_ids": ["read:users", "write:users"],
    "org_code": "<your-org-code>"
  }'

# Create a user-level key
curl -X POST https://your-domain.kinde.com/api/v1/api_keys \
  -H "Authorization: Bearer YOUR_M2M_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "User API Access",
    "api_id": "<your-api-id>",
    "scope_ids": ["read:users"],
    "user_id": "<your-user-id>"
  }'
```

## API key response

When you create a key via API, Kinde returns a `key` value. Store it securely - you won't be able to see it again!

You can also use the `id` to retrieve the key later.

```json
{
  "message": "API key created",
  "code": "API_KEY_CREATED",
  "api_key": {
    "id": "api_key_0195ac80a14e8d71f42b98e75d3c61ad",
    "key": "k_live_1234567890abcdef"
  }
}
```
