Testing is always a good idea after you start using the Kinde Management API, for example, after you add a new [machine to machine connection](/developer-tools/kinde-api/add-a-m2m-application-for-api-access/) for getting a Kinde API access token.

We recommend you test in a non-production environment.

The following procedure describes how to test using [Postman](https://postman.com/) - an API platform - but you can follow similar steps in your own app environment.

## Test the connection using various endpoints

You can test the API connection by sending a `GET` request to any Kinde API endpoint. See the [Kinde Management API library](https://kinde.com/api/docs/#kinde-management-api) for options.

Here’s an example using the `users listing` endpoint.

1. Create a new GET request.
2. Enter a URL that contains the `/users` endpoint, e.g. `https://<domain>.kinde.com/api/v1/users` .
