Kinde supports MCP (Model Context Protocol) so AI assistants like Cursor, Claude, and VS Code can call tools over a standard protocol. You can use MCP in two ways: expose **your** APIs to agents, or manage your **Kinde account** from an AI assistant.

In the Kinde UI, go to **Home > MCP Servers** to work with connections for your APIs.

## MCP connections for your APIs

<Aside>
  MCP connections are currently in beta. Please [contact support](https://kinde.com/support) to enable this feature for your account.
</Aside>

[MCP connections](/mcp-servers/add-mcp-connection/) turn your registered APIs into MCP tools that AI assistants call on behalf of your users.

- Each connection gets a unique URL: 
  ```
  https://<your_business>.kinde.com/mcp/<connection-id>
  ```
- Tools come from your API’s OpenAPI spec
- Clients authenticate with a **user API key**; scopes on that key control which tools appear
- Kinde proxies tool calls to your upstream API and records an audit entry

Use this when you want agents to call **your** product APIs — with auth, scopes, and auditing handled in Kinde.

## Manage your Kinde account with MCP

The [Kinde Management MCP server](/mcp-servers/manage-kinde-account-with-ai-assistants/) lets AI assistants interact with your Kinde account using a subset of the [Kinde Management API](/developer-tools/kinde-api/).

- Endpoint: `https://<your_subdomain>.kinde.com/mcp`
- Clients authenticate with an **Environment API key** (with MCP enabled)
- Tools cover users, organizations, roles, permissions, and related operations

Use this when you want to ask things like “What organizations do I have?” or “List users in the default organization” without leaving your IDE.

## Compare the two options

| | **MCP connections** (your APIs) | **Kinde Management MCP server** |
| --- | --- | --- |
| **Endpoint** | `https://<your_subdomain>.kinde.com/mcp/<connection-id>` | `https://<your_subdomain>.kinde.com/mcp` |
| **Purpose** | Expose **your** API operations as MCP tools | Manage your Kinde account from an AI assistant |
| **Auth** | User API key for the authorized API | Environment API key |
| **Tools source** | Your OpenAPI spec operations | Kinde Management API operations |
| **Where to configure** | Home → MCP Servers → Connections | Settings → Environment → APIs → Kinde Management API → API keys |
