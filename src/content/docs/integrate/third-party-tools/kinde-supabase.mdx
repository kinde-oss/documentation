User authentication and secure data access are critical components of modern web applications. Kinde simplifies the process by handling user authentication, managing session tokens, and offering robust user management features. 

When combined with Supabase's powerful database management and PostgreSQL’s Row-Level Security (RLS) policies, you can create a secure, scalable application that ensures user-specific data access.

In this guide, we will walk through integrating Kinde and Supabase. We’ll use Next.js as the application framework. You can then go on to build a to-do app following [this doc](/integrate/third-party-tools/kinde-supabase-todo-app/). 

## What you need

- A [Kinde](/get-started/guides/first-things-first/) account with **Admin** or **Engineer** permissions (sign up for free)
- A [Supabase](https://supabase.com/) account (sign up for free)

## Quickstart

### 1. Set up a Kinde application

1. Sign in to your Kinde dashboard and on the front page, select **Add application**.
2. Enter a name for the application (e.g., "Supabase") and select **Back-end web** as the application type. Select **Save**.
3. Select **Other back end** from the SDK list, and select **Save**. 
4. Go to the **Authentication** page and enable the authentication types you want your users to sign in with (e.g., Email + Password, Google, Facebook, etc.), and select **Save**.
5. Go to the **Details** page and copy the **Domain** (or [Custom domain](build/domains/pointing-your-domain/)), **Client ID** and **Client Secret**. We will need these in the next step.

### 2. Configure your Supabase project

1. Sign in to your [Supabase dashboard](https://supabase.com/).
2. If you don't have a project yet, create one by selecting **New Project** and finish the setup process.  
3. Go to **Authentication > CONFIGURATION > Sign In / Providers**
4. Scroll down to the **Custom Providers** section and select **New Provider**, a window opens.

    ![supabase custom auth provider screen 1](https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/b905dab8-efd7-441d-b055-4c5928f94f00/socialsharingimage)
5. Enter the following details:
    - **Provider Identifier**: `kinde`
    - **Display Name**: (e.g., "Kinde")
    - **Configuration Method**: Keep it default to **Auto-discovery (Recommended)**
6. Enter the **Issuer URL**: `https://<your_kinde_business>.kinde.com` or your [Custom domain](build/domains/pointing-your-domain/) if you have one.
7. Enter your application details you copied from Kinde from the previous step:
    - Client ID
    - Client Secret
    - Scopes: `openid, email, profile`

        <Aside>
        If you want to add the offline scope for getting a refresh token, use the `offline_access` scope for Kinde.
        </Aside>

    ![supabase custom auth provider screen 2](https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/66af62b7-dc62-4758-db12-81c1767c8f00/socialsharingimage)
8. Copy the **Callback URL** provided by Supabase. We will add it to Kinde.
9. Select **Update provider**.
10. Go to your Kinde application > **Details** page and enter the callback URL you copied from Supabase to the **Allowed Callback URLs** field. Select **Save**.

    ![supabase kinde callback url screen](https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/43fe4fd7-e12c-4c8b-7b92-56ab47a62100/socialsharingimage)

### 3. Add Kinde authentication to your Supabase app

You can add Kinde authentication to your Supabase app by using the `signInWithOAuth` method.

```typescript title="JavaScript example"
const { data, error } = await supabase.auth.signInWithOAuth({
  provider: 'custom:kinde',
});
```

Check out the [Supabase documentation](https://supabase.com/docs/guides/auth/custom-oauth-providers) for more details.

Here's a guide on how to add Kinde authentication to a Next.js app with Supabase:

1. If you want to create a new Supabase Next.js project run the following command:

    ```bash
    npx create-next-app -e with-supabase
    ```

2. Add the following to your `.env.local` file:

    ```env
    # env.local
    NEXT_PUBLIC_KINDE_DOMAIN=https://<your-subdomain>.kinde.com
    ```

    You can use either your Kinde domain (`https://your_brand.kinde.com`) or your [Custom domain](build/domains/pointing-your-domain/) (`https://auth.yourbrand.com`). Make sure you use the same one you used during the OIDC configuration in the previous step.

3. Create the required files:

    ```bash
    mkdir -p app/auth/callback
    touch components/login-button.tsx
    touch components/register-button.tsx
    touch app/auth/callback/route.ts
    ```

    `components/logout-button.tsx` already exists in the starter — modify it (see step 8 below), don't create it.

4. Open `components/login-button.tsx`, enter the following code, and save the file:

    ```tsx
    "use client";

    import { createClient } from "@/lib/supabase/client";
    import { Button } from "@/components/ui/button";

    export function LoginButton() {
      const login = async () => {
        const supabase = createClient();
        await supabase.auth.signInWithOAuth({
          provider: "custom:kinde",
          options: {
            redirectTo: `${window.location.origin}/auth/callback`,
          },
        });
      };

      return (
        <Button size="sm" variant="default" onClick={login}>
          Sign in
        </Button>
      );
    }
    ```

5. Open `components/register-button.tsx`, enter the following code, and save the file:

    ```tsx
    "use client";

    import { createClient } from "@/lib/supabase/client";
    import { Button } from "@/components/ui/button";

    export function RegisterButton() {
      const register = async () => {
        const supabase = createClient();
        await supabase.auth.signInWithOAuth({
          provider: "custom:kinde",
          options: {
            queryParams: {
              prompt: "create",
            },
            redirectTo: `${window.location.origin}/auth/callback`,
          },
        });
      };

      return (
        <Button size="sm" variant="default" onClick={register}>
          Sign up
        </Button>
      );
    }
    ```

6. Open `app/auth/callback/route.ts`, enter the following code, and save the file:

    ```ts
    import { createClient } from "@/lib/supabase/server";
    import { NextResponse } from "next/server";

    export async function GET(request: Request) {
      const { searchParams, origin } = new URL(request.url);
      const code = searchParams.get("code");
      const next = searchParams.get("next") ?? "/protected";

      if (code) {
        const supabase = await createClient();
        const { error } = await supabase.auth.exchangeCodeForSession(code);
        if (!error) {
          return NextResponse.redirect(`${origin}${next}`);
        }
      }

      return NextResponse.redirect(`${origin}/auth/error?error=oauth_callback_failed`);
    }
    ```

7. Open `components/auth-button.tsx` and replace the login/sign-up page links with OAuth buttons:

    - Remove the `Link` imports and `<Link href="/auth/login">` / `<Link href="/auth/sign-up">` buttons.
    - Add the following imports:

        ```tsx
        import { LoginButton } from "./login-button";
        import { RegisterButton } from "./register-button";
        ```

    - Update the logged-out state to render:

        ```tsx
        <div className="flex gap-2">
          <LoginButton />
          <RegisterButton />
        </div>
        ```

    - Keep the logged-in state the same — show the user email and `<LogoutButton />`.

8. Open `components/logout-button.tsx` and replace `router.push("/auth/login")` with Supabase sign-out and a Kinde logout redirect. Replace the entire file content with the following:

    ```tsx
    "use client";

    import { createClient } from "@/lib/supabase/client";
    import { Button } from "@/components/ui/button";

    export function LogoutButton() {
      const logout = async () => {
        const supabase = createClient();
        await supabase.auth.signOut();
        const kindeDomain = process.env.NEXT_PUBLIC_KINDE_DOMAIN;
        const redirect = encodeURIComponent(window.location.origin);
        window.location.href = `${kindeDomain}/logout?redirect=${redirect}`;
      };

      return <Button onClick={logout}>Logout</Button>;
    }
    ```

9. Open `lib/supabase/proxy.ts` and change the unauthenticated redirect from `/auth/login` to `/`:

    ```ts
    url.pathname = "/";
    ```

10. Open `app/protected/page.tsx` and change the unauthenticated redirect from `/auth/login` to `/`:

    ```ts
    redirect("/");
    ```

### 4. Test user authentication

1. Run the development server:

    ```bash
    npm run dev
    ```

2. Go to http://localhost:3000 and select the **Sign up** button. You will be redirected to the Kinde hosted registration page.

3. Complete the registration process and you will be redirected to your Supabase app protected page.

    ![supabase kinde app protected page screen](https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/2f7698b3-63d7-4263-a0cd-5a64bb1f4c00/socialsharingimage)

4. Go to your Kinde dashboard > **Users** and you will see the new user in the list.

5. Go to your Supabase dashboard > **Authentication > MANAGE > Users** and you will also see the same users in the list.

    ![supabase kinde user management screen](https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/4056bfe1-e8d4-46cb-af36-da87d4fbbc00/socialsharingimage)
