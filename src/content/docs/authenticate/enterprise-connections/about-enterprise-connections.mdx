Enterprise authentication is a common method for managing user access to systems in large organizations.

Kinde supports a number of enterprise connection types, including:

- [Custom SAML](/authenticate/enterprise-connections/custom-saml/)
- [Microsoft Entra ID](/authenticate/enterprise-connections/azure/) (was Azure AD) WS Federated or Open ID
- [Google workspace](/authenticate/enterprise-connections/custom-saml-google-workspace/) (via SAML)
- [Okta](/authenticate/enterprise-connections/okta-saml-connection/) (via SAML)

<Aside type="upgrade">

The number of enterprise connections you can have depends on your [Kinde plan](https://kinde.com/pricing/).

</Aside>

## Option to trust emails from providers

There's no guarantee that the email kept on file with an enterprise auth provider remains current. Because of this, Kinde gives you the option to 'trust' provider emails. For more information, see [User identity and verification](/authenticate/about-auth/identity-and-verification/).

## Just-in-time provisioning

When a user tries to sign in using your SAML or Entra ID enterprise connection, and they do not exist, you can choose to add them automatically. Select this option when you configure the connection.

![image of switch](https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/74717fdb-d6d2-4f0b-dcf6-2aef95cb7600/public)

## Extended attributes for MS Entra ID

You can include **Extended profiles** and **User groups** in authentication tokens for MS Entra ID connections. Select the relevant options when you configure the connection.

![Image of switch](https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/bc92867f-2fc8-4676-2019-ce4b81783c00/public)

## MS Entra ID profile sync

When you configure the MS Entra ID connection, you can choose whether to sync user profiles each time a user signs in. This is a recommended way to keep user profiles up to date.
